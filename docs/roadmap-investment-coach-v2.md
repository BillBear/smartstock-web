# SmartStock 投资教练 V2 开发计划

## 目标

把当前“智能选股榜单”升级为“小白可执行的自动选股与交易教练系统”。系统必须回答：今天是否交易、买什么、怎么买、错了怎么办、策略历史是否可信。

## 阶段 1：交易计划化与可信度标识

目标：
- 智能选股从榜单改为“今日交易计划”。
- 每只股票输出决策等级 A/B/C/禁入。
- 明确标注当前上涨概率/回撤概率是否为“规则代理概率”。
- 推荐结果必须展示策略证据状态：实盘准入、模拟验证、未验证。
- 策略不满足准入时，页面默认建议模拟验证，不暗示可直接实盘。

验收：
- `/api/coach/picks/today` 返回 `trade_plan`、`decision`、`probability_model`。
- `/smart-screen` 首屏展示今日行动，而不是只展示表格。
- 构建通过，接口可返回推荐，页面无 404。

## 阶段 2：推荐落库、回测证据与模拟复盘闭环

目标：
- 每次推荐生成批次落库，保存策略版本、数据快照、推荐理由和风险。
- 自选股/模拟买入绑定推荐快照，不再依赖临时内存。
- 策略回测结果成为推荐准入门槛。
- 模拟交易结果形成复盘样本。

验收：
- 推荐详情可从数据库恢复。
- 重启服务后，自选股仍能看到原始推荐分、策略、理由和风险。
- 回测未通过时，智能选股显示“仅模拟验证”或“不交易”。

## 阶段 3：历史样本概率模型与校准

目标：
- 构建历史训练样本：特征、未来收益标签、未来回撤标签。
- 用真实历史样本训练上涨概率和回撤概率模型。
- 输出概率校准指标：Brier Score、ECE、分层命中率。
- 替换当前规则代理概率，或至少支持模型概率与代理概率并行对比。

验收：
- 概率接口返回模型版本、训练区间、校准指标。
- 高概率组历史命中率必须高于低概率组。
- 页面明确展示“模型概率”或“代理概率”。

## 阶段 4：小白投资教练体验

目标：
- 新增“今日是否出手”体验：买入、观察、不交易。
- 新增“持仓教练”：止损、止盈、失效条件、复盘提醒。
- 新增“每周复盘”：胜率、收益、亏损原因、下周建议。
- 每个推荐附带教学卡片，让用户理解选股逻辑。

验收：
- 小白用户不需要理解所有指标，也能看到明确行动建议。
- 模拟交易亏损时，系统能给出归因：策略失效、执行偏差、市场环境变化、个股事件冲击。
- 每周复盘页面可展示真实模拟记录统计。
