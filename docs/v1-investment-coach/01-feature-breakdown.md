# 01. 功能拆解（V1）

## 1. 产品目标

把 SmartStock 做成面向个人投资者的"决策系统"，而不是单纯行情软件。

V1 成功标准：

1. 每个交易日输出 3-5 只可执行标的（最多 10 只）。
2. 每只标的具备完整决策信息：盈利概率、风险概率、理由、仓位、退出规则。
3. 每条推荐都可追溯到策略回溯证据（样本外）。
4. 用户可按推荐直接执行，并在复盘中理解选股逻辑。

---

## 2. 核心业务闭环

1. 市场全景：判断当前市场状态（进攻/均衡/防守）。
2. 智能筛选：在当前状态下生成"今日可投池"。
3. 策略回溯：为每个策略/推荐提供历史有效性证据。
4. 用户执行：加入观察或模拟组合，按计划管理仓位。
5. 每周复盘：对命中和失误做结构化归因，沉淀选股技能。

---

## 3. 模块拆解

## 3.1 市场全景（/dashboard）

### 定位

给系统和用户一个"市场环境标签"，用于控制当天选股门槛与仓位。

### V1 功能

1. 市场状态评分（0-100）
- 生成 `state_tag`：`offensive` / `neutral` / `defensive`
- 生成 `state_confidence`：`high` / `medium` / `low`

2. 四类驱动指标
- 趋势：主指数相对 MA20/MA60
- 宽度：上涨家数比、新高新低比
- 资金：北向/主力净流向
- 风险：波动率、回撤压力

3. 环境结论输出
- 今日建议仓位区间（例如 20%-40%）
- 风险提示（2-3 条）
- 对智能筛选的动态门槛（提/降阈值）

### 用户价值

用户不用盯盘，也知道今天是该进攻还是防守。

---

## 3.2 智能筛选（/smart-screen）

### 定位

从"评分榜单"升级为"可执行决策单"。

### V1 功能

1. 自动候选生成
- 股票池清洗：剔除 ST、停牌、流动性过低
- 策略触发：
  - S1 趋势突破 + 资金确认
  - S2 回调反转 + 量价修复

2. 概率输出（核心）
- `up_prob`：`P(未来15日收益 > +8%)`
- `dd_prob`：`P(未来15日最大回撤 > 6%)`
- `confidence_level`：概率可信度标签

3. 决策参数输出
- `action`：buy / watch / pass
- `entry_range`
- `take_profit`
- `stop_loss`
- `position_pct`
- `invalid_conditions`

4. 理由与风险解释
- `reasons`（最多 3 条）
- `risks`（最多 3 条）
- `evidence_summary`（策略表现摘要）

5. 用户执行动作
- 一键加入观察
- 一键加入模拟组合
- 标记已执行/未执行（用于后续学习反馈）

### 用户价值

直接拿到"今天能不能买、怎么买、错了怎么办"。

---

## 3.3 策略回溯（/backtest）

### 定位

从"回测展示"升级为"推荐证据中心 + 策略实验台"。

### V1 功能

1. 证据中心（面向推荐）
- 策略样本外胜率
- 年化收益、最大回撤、夏普、盈亏比
- 分市场状态表现（offensive/neutral/defensive）

2. 策略实验台（面向进阶用户）
- 参数可配置：持有天数、止盈止损、入选阈值
- 成本模型：手续费、印花税、滑点
- 交易约束：涨跌停、停牌、流动性

3. 回溯方式
- Walk-forward 样本外验证（训练36个月，测试6个月，滚动）
- 参数敏感性检查（防止过拟合）

### 用户价值

知道推荐"为什么可信"，也知道在哪些市场不该用。

---

## 4. 用户体验主流程

1. 用户打开 `/smart-screen`。
2. 系统展示"今日市场状态"与仓位建议。
3. 系统展示 3-5 只"今日可投"。
4. 用户查看单票决策详情：概率 + 理由 + 风险 + 执行规则。
5. 用户执行并记录。
6. 每周收到复盘：命中率、失误原因、下周建议。

---

## 5. V1 边界与规则

1. 频率：日更（收盘后生成，次交易日执行）。
2. 持有周期：5-20 交易日。
3. 推荐数量：默认 3-5，最大 10。
4. 风险控制：
- 单票仓位上限 10%
- 行业集中度上限 30%
- 防守市场可输出"今日不交易"

---

## 6. 关键指标（KPI）

## 产品指标

1. 每日推荐生成成功率 > 99%
2. 决策单查看率
3. 决策执行率
4. 周复盘阅读率

## 投资指标

1. 推荐组合样本外超额收益（相对基准）
2. 最大回撤
3. 胜率与盈亏比

## 概率质量指标

1. Brier Score
2. ECE（校准误差）
3. 分层命中稳定性（高概率组命中率应更高）

---

## 7. 上线验收标准（业务）

1. 每条推荐必须同时包含：
- 盈利概率
- 风险概率
- 入场/止损/止盈
- 推荐理由
- 风险点
- 回溯证据摘要

2. 不满足策略门槛时，允许输出"今日不交易"。

3. 任意推荐必须可追溯：
- 数据版本
- 模型版本
- 策略版本
- 生成批次

---

## 8. 实施里程碑

1. M1（1周）：数据模型 + 离线任务骨架 + 推荐结果表落库
2. M2（1周）：接口联调（今日可投、详情、证据）
3. M3（1周）：页面改造（Dashboard/SmartScreen/Backtest）
4. M4（1周）：灰度验证与复盘报告
